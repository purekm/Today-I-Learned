우선순위 큐의 구현방법 3가지
배열을 기반으로 구현
연결 리스트를 기반으로 구현
힙을 이용하는 방법

배열을 기반으로 구현
-> 우선순위가 높은 데이터 반환은 쉽지만 삭제 및 삽입이 비효율적 O(n)

연결 리스트를 기반으로 구현
-> 배열과 마찬가지로 우선순위가 낮은 데이터 삭제 및 삽입이 비효율적 O(n)

=> 힙으로 구현(완전 이진 트리) O(log n)
부모 노드 데이터의 우선순위가 자식 노드 데이터 순위보다 높음 

우선순위 큐의 구현-> 힙을 이용
힙의 구현-> 연결리스트 or 배열 => 배열을 통해 구현
연결리스트를 기반으로 힙을 구현시, 새로운 노드를 힙의 마지막 위치에 추가하는 것이 쉽지않음

배열은 인덱스를 통해서 쉽게 접근 가능
인덱스와 우선순위를 알기쉽게하기 위하여 인덱스0은 비워두고 만든다.
