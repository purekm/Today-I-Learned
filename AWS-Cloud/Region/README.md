고가용성 및 내결함성
- 데이터센터 하나가 정전되었을 때, 서비스가 끊기지 않도록 다른 곳으로 분산시켜줌
- AWS에서는 리전을 사용

리전
- 비즈니스 트래픽이 많은 곳에 리전 설치
- 앱이 실행하는데 필요한 모든 서비스가 구비된 다양한 데이터 센터가 존재
- 각 리전은 AWS의 고속 광섬유 네트워크를 통해 다른 리전에 연결
- 각 리전의 데이터는 독립적인 상태로, 데이터 내보내기를 요청하지 않는 한 리전을 벗어나지 않음
- 각 리전은 여러 데이터센터(AZ)로 구성됨
- 리전의 2개 이상의 가용 영역에서 실행(EC2 서버를 2개이상 실행한다는 의미)
- 리전 수준에서 실행된다는 의미는 사용자가 따로 신경쓰지 않아도 여러AZ, 여러 가용영역에서 서비스가 동시에 실행된다는 뜻

고려할 사항 4가지
규정 준수
- 각 리전은 지역 법률과 국가 법령이 적용됨

근접성
- 지연시간을 생각해서 고객이 많은 곳에 리전을 선택하는 것이 좋을지도 모름

기능 가용성
- 내 리전에서는 원하는 AWS 기능이 없을 수 있음

요금
- 같은 서비스가 국가마다 가격이 다를 수 있음

엣지 로케이션
- 데이터 복사본 캐싱
- CDN(Amazon Cloud Front)는 전 세계 고객에게 데이터를 짧은 지연시간으로 전달해주는 서비스
- 전세계 리전에 있는 데이터를 엣지 로케이션에 푸시할 수 있음
- DNS(Amazon Route 53)도 실행함

AWS Outposts
- 사용자의 데이터 센터 내부에 소형 리전을 설치하는 것
- 기업 내 자체 건물에서 AWS 서비스를 사용하고싶을 때 사용
